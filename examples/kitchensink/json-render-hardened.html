JSON Render Hardened API & Components

Covers the Phase 1.6 programmatic API (replaceSpec(), toolUseId) and the Phase 1.7 component completeness enhancements (CodeBlock copy + line numbers, Checklist counter, TextField char counter) added under COMP-14.

AC #1 — element.replaceSpec(obj) re-renders the DOM

The programmatic helper installed in element.js:setup() must stringify the spec, assign it to textContent, and trigger a re-render via the MutationObserver-driven reactivity path.

AC #2 — replaceSpec accepts pre-serialized JSON string

Passing a string should assign directly (no double-encoding). Verifies the typeof === 'string' branch.

AC #3 — element.toolUseId getter/setter mirrors dataset

Setting element.toolUseId = "tu_xyz" must stamp data-tool-use-id on the host and the property must read back the same value. Setting null/empty removes the attribute.

AC #4 — CodeBlock renders a copy button in the header

The new .jr-codeblock-copy button must appear in the header and be clickable. Initial label is "Copy".

AC #5 — CodeBlock showLineNumbers renders the gutter

Setting props.showLineNumbers: true must render a .jr-codeblock-lines gutter containing one line number per line of code.

AC #6 — CodeBlock has no gutter when showLineNumbers is absent

Backwards compatibility: existing CodeBlock consumers that do not opt in must render without the line gutter.

AC #7 — Checklist renders a completion counter

A checklist with 2 of 5 items checked must render the .jr-checklist-counter div with the text 2/5 complete.

AC #8 — Empty Checklist does NOT render a counter

A checklist with zero items must skip the counter entirely (avoids "0/0 complete" clutter).

AC #8b — Clicking a Checklist item updates the visual state + counter

With every spec item set to checked: false: (a) zero boxes render checked on first paint, (b) a click on item 0 flips that box to checked, adds the .checked class to its label (enabling the strikethrough CSS rule), updates the counter to 1/3 complete, and fires a bubbling jr-action with {index:0, checked:true, label:'Milk'}, (c) a second click flips it back to unchecked and the counter returns to 0/3 complete.

AC #9 — TextField with maxLength renders a counter

The .jr-textfield-counter span must appear when maxLength is set, with initial text 0/<max>.

AC #10 — TextField counter updates on input

Typing into the input must live-update the counter's text via the oninput handler.

AC #11 — TextField without maxLength has NO counter

Backwards compatibility: TextFields that do not declare a maxLength must render without the counter span.

AC #12 — CodeBlock copy button success path (clipboard resolves)

Clicking the copy button must call navigator.clipboard.writeText and flip the button label to "Copied!" briefly.

AC #13 — CodeBlock copy button failure path (clipboard rejects)

When navigator.clipboard.writeText rejects (e.g. insecure context, permission denied), the button must flip to "Failed" via the .catch branch instead of throwing.

AC #14 — Disconnecting the element cleans up the programmatic API

Removing a <json-render> from the DOM fires hyper-element's disconnectedCallback, which runs the teardown returned by setup(). That teardown must delete both replaceSpec and toolUseId from the element so no stale references survive.
