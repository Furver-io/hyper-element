json-render Coverage Specs

Programmatic test suite designed strictly to exercise edge cases, branch fallbacks, error emissions, and missing attributes across the json-render core validators and renderers.

Validator Errors

Intentionally triggers registration errors and schema validation failures to ensure they are handled gracefully.

// Hit registerComponent errors
try { window.registerComponent('', () => {}); } catch(e) { }
try { window.registerComponent('Valid', null); } catch(e) { }

// Override-of-built-in warning path: registerComponent('Text', ...) where 'Text'
// is a built-in. The warning is logged to console; functionally the registry
// happily accepts the override (it's a deliberate but uncommon op).
window.registerComponent('Text', (Html, def, key) =>
  Html.wire(def, ':' + key)`${def.props?.content ?? ''}`
);

// Hit validateSpec errors
window.validateSpec(null);
window.validateSpec({ });
window.validateSpec({ root: 'a' });
window.validateSpec({ root: 'a', elements: { } });

// Rules
window.validateSpec({
  root: 'a',
  elements: {
    // 'ghost' triggers Rule 2 dangling-child reference
    'a': { type: 'Row', children: ['b', 'c', 'ghost'] },
    'b': { type: 'UnknownType' }, // Rule 4
    'd': { type: 'Button' }, // Rule 5 missing action
    'e': { type: 'Progress', props: { value: 150 } }, // Rule 6
    'f': { type: 'Checklist', props: { items: [{ label: 1, checked: 'yes' }] } }, // Rule 7 (bad item shape)
    // Rule 7 (items not an array at all) — separate Checklist with a string instead of array
    'g': { type: 'Checklist', props: { items: 'not-an-array' } },
    'c': { type: 'Card', children: ['a'] } // Cycle
  }
}, new Set(['CustomVal']));

Components Fallback Coverage

Exercises all internal components stripped of any props or attributes to ensure null-safeties trigger across all default configurations.

const Html = { wire: () => () => 'wired' };
const types = ['Alert', 'Button', 'Card', 'Checklist', 'CodeBlock',
  'Column', 'Divider', 'Image', 'Progress', 'Row', 'TextField', 'Text'];
for (const type of types) {
  const fn = window.BUILT_IN_COMPONENTS.get(type);
  if (fn) {
    try { fn(Html, null, 'key'); } catch(e) {}
    try { fn(Html, { type }, 'key'); } catch(e) {}
  }
}

renderSpec throws when root key is missing

When a spec sets root to a key that does not exist in elements, the core renderer (renderSpecTree) throws. The <jr-ui> component catches the throw and renders an inline error alert. This exercises the throw branch in src/json-render/renderer.js lines 84-86. 

<jr-ui id="missing-root-target" data-spec='{"root":"ghost","elements":{"real":{"type":"Text","props":{"content":"present but unreachable"}}}}'></jr-ui>

listComponentTypes() public API

The listComponentTypes() export returns a snapshot of every registered component type — built-ins plus any added via registerComponent(). Useful for runtime introspection, debug tools, and self-documenting validators. 

// Snapshot before any custom registration: should contain only the built-ins.
const before = window.listComponentTypes();

// Register a custom component, then snapshot again.
window.registerComponent('MyDebugWidget', (Html, def, key) =>
  Html.wire(def, ':' + key)`debug`
);
const after = window.listComponentTypes();

// Verify both snapshots and the registration delta.
const section = document.querySelector('[data-test="list-component-types"]');
const builtIns = ['Card', 'Row', 'Column', 'Button', 'Text', 'Alert',
  'Progress', 'Divider', 'CodeBlock', 'Image', 'Checklist', 'TextField'];
const allBuiltInsPresent = builtIns.every((t) => before.includes(t));
const customAddedAfter = after.includes('MyDebugWidget');
const customAbsentBefore = !before.includes('MyDebugWidget');
const isArray = Array.isArray(before) && Array.isArray(after);
const sizeDelta = after.length === before.length + 1;

document.getElementById('list-types-output').textContent =
  `before(${before.length}): ${before.join(', ')}\nafter(${after.length}): ${after.join(', ')}`;

section.dataset.testResult =
  (isArray && allBuiltInsPresent && customAbsentBefore && customAddedAfter && sizeDelta)
    ? 'pass'
    : 'fail';
