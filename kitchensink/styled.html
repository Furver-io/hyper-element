+styled Inline Styling System

Demonstrates the +styled template syntax for react-outline-inspired inline styling with base styles, prop flags, logic functions, and color palettes.

1. Base Styles

Simple base styles keyed by tag name.

hyperElement('styled-base', {
  styled: [{
    h2: { color: 'red', fontSize: '24px' },
    span: { fontWeight: 'bold' }
  }],
  render: (Html) => Html`
    <h2+styled>Title</h2>
    <span+styled>Content</span>
  `
});

2. Shared Selectors

Comma-separated selectors apply to multiple tags.

hyperElement('styled-shared', {
  styled: [{
    'h3, p': { margin: '0', fontFamily: 'Arial' }
  }],
  render: (Html) => Html`
    <h3+styled>Heading</h3>
    <p+styled>Paragraph</p>
  `
});

3. Prop Flags (Boolean Variants)

Toggle visual states with boolean attributes.

hyperElement('styled-flags', {
  styled: [{
    div: {
      base: { padding: '10px', border: '1px solid gray' },
      error: { borderColor: 'red', backgroundColor: '#fee' },
      large: { padding: '20px', fontSize: '18px' }
    }
  }],
  render: (Html) => Html`
    <div+styled>Normal</div>
    <div+styled error>Error</div>
    <div+styled error large>Error + Large</div>
  `
});

4. Logic Functions

Compute styles dynamically based on state.

hyperElement('styled-logic', {
  styled: [
    { span: { display: 'inline-block' } },
    { span: (styleValue) => ({ width: `${styleValue}%` }) }
  ],
  render: (Html, ctx) => Html`
    <span+styled style=${ctx.attrs.percent || 50}>Bar</span>
  `
});

5. Color Palettes (withOptions)

Reference named colors from a shared palette.

const themed = withOptions({
  colors: { primary: '#007bff', danger: '#dc3545' }
});

themed('styled-themed', {
  styled: [{
    button: { backgroundColor: 'primary', color: 'white' }
  }],
  render: (Html) => Html`<button+styled>Click</button>`
});

6. Style Passthrough

When no logic function exists, style object merges directly.

hyperElement('styled-passthrough', {
  styled: [{
    div: { display: 'flex', gap: '8px' }
  }],
  render: (Html) => Html`
    <div+styled style=${{ flexDirection: 'column' }}>Content</div>
  `
});

7. No Styles Defined for Tag

Elements with +styled but no matching styles render without style attribute. 

hyperElement('styled-no-match', {
  styled: [{
    h2: { color: 'blue' }
  }],
  render: (Html) => Html`<span+styled>No span styles</span>`
});

8. Mixed +styled and Regular Elements

Not all elements need styling - only +styled elements get styles.

hyperElement('styled-mixed', {
  styled: [{
    p: { color: 'navy' }
  }],
  render: (Html) => Html`
    <p+styled>Styled paragraph</p>
    <p>Regular paragraph</p>
  `
});

9. Logic Function with Context Access

Logic functions receive (styleValue, ctx, store).

hyperElement('styled-context', {
  styled: [
    { div: { padding: '10px' } },
    { div: (val, ctx) => ({ color: ctx.attrs.textcolor || 'black' }) }
  ],
  render: (Html) => Html`<div+styled>Context-aware</div>`
});

10. Color in Compound Value

Colors within strings are also resolved.

const themed2 = withOptions({
  colors: { primary: '#007bff' }
});

themed2('styled-compound', {
  styled: [{
    div: { border: '2px solid primary' }
  }],
  render: (Html) => Html`<div+styled>Border</div>`
});

11. withOptions Without Tag Name

Returns class for manual registration.

const factory = withOptions({ colors: { accent: '#ff6600' } });
const MyClass = factory({ ... });  // No tag
customElements.define('styled-manual', MyClass);

12. withOptions Render Function Shorthand

Passing just a render function to the factory (coverage for shorthand path).

themed('styled-shorthand', (Html) => Html`...`);

13. Non-String Values with Color Palette

Numbers pass through unchanged even when color palette is active.

withOptions({ colors: { accent: '#ff0' } })
styled: [{ div: { zIndex: 100, opacity: 0.5, color: 'accent' } }]

14. Null/Empty Values Clear Properties

Null values remove the property from the style.

render: (Html) => Html`<div+styled style=${{ color: null }}>...</div>`

15. Empty Styled Config

Works correctly when styled array is empty or has no matching styles.

styled: [{}]  // Empty base styles

16. Re-render Updates Styles

Styles update when component re-renders (DOM traversal fallback).

Re-renders with different style value

17. Direct Render Fallback

Using Html.lite directly outside a component still applies inline styles.

// Direct render (not in a component)
Html.lite(target)`<div+styled style=${{ color: 'green' }}>...</div>`

18. +styled Without Styled Config

Using +styled syntax without defining styled config passes through styles.

hyperElement('no-styled-config', {
  // No styled array defined
  render: (Html) => Html`<div+styled style=${{ color: 'blue' }}>...</div>`
});

19. +styled with Empty Styled Array

Empty styled array still allows style passthrough.

styled: [],  // Empty array
render: (Html) => Html`<div+styled style=${{ margin: '10px' }}>...</div>`

20. +styled Without Config or Inline Style

Covers the return null fallback path when no styled config and no style value.

// No styled config, no inline style
hyperElement('no-config-no-style', {
  render: (Html) => Html`<span+styled>Plain text</span>`
});

21. Conditional Prop Flag via Style Object

Like react-outline: <Title style={{ error: true, width: '200px' }}>

hyperElement('styled-conditional-flag', {
  styled: [{
    div: {
      base: { padding: '10px', border: '1px solid gray' },
      error: { borderColor: 'red', backgroundColor: '#fee' }
    }
  }],
  render: (Html, ctx) => Html`
    <div+styled style=${{ error: ctx.attrs.haserror === 'true', width: '200px' }}>Conditional</div>
  `
});

22. Multiple Conditional Flags via Style Object

Pass multiple boolean flags via style object: style=${{ error: true, large: true }}

hyperElement('styled-multi-flags', {
  styled: [{
    div: {
      base: { padding: '10px', fontSize: '14px' },
      error: { color: 'red' },
      large: { fontSize: '24px', padding: '20px' }
    }
  }],
  render: (Html, ctx) => Html`
    <div+styled style=${{ error: ctx.attrs.err === 'true', large: ctx.attrs.big === 'true' }}>Multi</div>
  `
});

23. Logic Function Conditional Styling

Logic functions receive (styleValue, ctx, store) and return computed styles.

hyperElement('styled-logic-conditional', {
  styled: [
    { span: { color: 'gray' } },
    { span: (important) => ({
        fontSize: important ? '24px' : '14px',
        fontWeight: important ? 'bold' : 'normal'
      })
    }
  ],
  render: (Html) => Html`
    <span+styled style=${true}>Important</span>
    <span+styled style=${false}>Normal</span>
  `
});

24. Function Without Base Style

Like react-outline: Logic function exists but no base style defined.

hyperElement('styled-fn-no-base', {
  styled: [
    { /* no 'div' here */ },
    { div: () => ({ backgroundColor: 'lightgray', padding: '15px' }) }
  ],
  render: (Html) => Html`<div+styled>Function Only</div>`
});

25. External CSS Class Integration

Like react-outline: className works alongside +styled.

hyperElement('styled-with-class', {
  styled: [{
    div: { padding: '20px', backgroundColor: '#e0e0ff' }
  }],
  render: (Html) => Html`
    <div+styled class="custom-animation">Styled + Class</div>
  `
});

26. Store Access in Logic Function

+styled exclusive: Logic functions receive (styleValue, ctx, store).

hyperElement('styled-store-access', {
  setup: (ctx, onNext) => onNext({ loading: true }),
  styled: [
    { div: { padding: '15px' } },
    { div: (val, ctx, store) => ({
        opacity: store.loading ? '0.5' : '1',
        cursor: store.loading ? 'wait' : 'default'
      })
    }
  ],
  render: (Html, store) => Html`<div+styled>Loading State</div>`
});

27. Implied Tag Names

Like react-outline: Style names matching HTML tags get auto-applied.

hyperElement('styled-implied-tags', {
  styled: [{
    table: { borderCollapse: 'collapse', width: '100%' },
    tr: { borderBottom: '2px solid #333' },
    td: { padding: '8px', textAlign: 'center' },
    button: { cursor: 'pointer', backgroundColor: '#007bff', color: 'white' }
  }],
  render: (Html) => Html`
    <table+styled>
      <tr+styled><td+styled>A</td><td+styled>B</td></tr>
    </table>
    <button+styled>Click</button>
  `
});

28. Complex Nested Structure

Multiple nested elements with different styles applied correctly.

hyperElement('styled-complex-nested', {
  styled: [{
    article: { border: '1px solid #ccc', borderRadius: '8px', overflow: 'hidden' },
    header: { backgroundColor: '#f0f0f0', padding: '10px' },
    h1: { margin: '0', fontSize: '18px', color: '#333' },
    section: { padding: '15px' },
    p: { margin: '0', lineHeight: '1.5', color: '#666' },
    footer: { backgroundColor: '#fafafa', padding: '10px', textAlign: 'right' }
  }],
  render: (Html) => Html`
    <article+styled>
      <header+styled><h1+styled>Title</h1></header>
      <section+styled><p+styled>Content here</p></section>
      <footer+styled>Footer</footer>
    </article>
  `
});

29. Parser Hooks API

Coverage test for parser-hooks.js utilities exported for external consumers.

detectStyledSuffix('div+styled')  // { isStyled: true, tagName: 'div' }
detectStyledSuffix('div')         // { isStyled: false, tagName: 'div' }

// isValidPropFlag - validates prop flag names
